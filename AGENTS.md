# Agent Instructions

## Repository Ownership

Velocity Architecture owns:

- enterprise design
- enterprise architecture
- business architecture
- solution architecture
- architecture governance
- public reference models
- master templates
- methodology
- international, Australian, New Zealand and APAC alignment guidance
- framework-neutral evidence baseline
- public thought leadership

## Must Not Absorb

- client-specific artefacts
- runtime policy enforcement
- credentials
- private Ordo Animi implementation
- claims of universal certification or automatic compliance

## Permitted Change Types

- Constitutional documentation
- Repository-specific governance records
- Markdown and JSON schema documentation approved for this repository
- Small corrective edits tied to validation findings

## Prohibited Change Types

- Application code during v0.1
- Cross-repository changes without explicit instruction
- Package managers, dependencies or generated lockfiles during v0.1
- GitHub Actions or deployment configuration during v0.1
- Authentication, storage services, cloud integrations or automated writes during v0.1
- Claims of deployment, compliance, validation or certification without evidence

## Data Restrictions

Do not commit real client data, personal information, credentials, secrets, tokens, production identifiers, or unapproved operational records.

## Security Requirements

Keep the repository private by default, least-privilege, and secret-free. Do not place credentials in examples.

## Required Validation

Before marking work complete, validate remotes, clean tree, required files, JSON, JSON Schemas, Markdown links, credentials, package files, executable scripts, GitHub Actions, deployment configuration and unintended application code.

## Founder / CTO Routing

Stop and request Founder / CTO routing if ownership, data handling, security posture, external integration, release status or implementation authority is unclear.
