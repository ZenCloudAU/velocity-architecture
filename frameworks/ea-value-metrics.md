# Velocity EA Value Metrics

## Purpose

This document distinguishes architecture capability maturity, enterprise architecture value metrics, and Velocity decision metrics.

## Three Measurement Layers

| Layer | Core question | Example |
|---|---|---|
| ACMM | How mature is the EA capability? | Are processes, roles, standards, and governance defined? |
| EA Metrics | What impact does EA create? | Does EA reduce risk, improve reuse, align spend, and support strategy? |
| Velocity Decision Metrics | How do decisions improve speed, quality, traceability, and execution? | Are decisions faster, clearer, traceable, and connected to delivery? |

## ACMM

ACMM measures architecture capability maturity:

- governance process maturity
- standards maturity
- role clarity
- repository maturity
- architecture method adoption
- compliance and review process

ACMM can show whether the architecture function is structured, but it does not prove that architecture is improving enterprise decisions.

## EA Value Metrics

EA Metrics measure business impact:

- cost avoidance
- risk reduction
- reuse
- simplification
- delivery acceleration
- stakeholder satisfaction
- strategy alignment
- technology rationalisation

EA Metrics help executives see the value created by architecture work.

## Velocity Decision Metrics

Velocity Decision Metrics measure whether architecture improves:

- decision speed
- decision quality
- decision traceability
- execution alignment
- artefact usefulness
- governance throughput
- AI accountability
- PMO integration

## High Maturity / Low Value Trap

An organisation can have:

- mature architecture process
- many standards
- detailed artefacts
- governance boards
- repositories

and still produce low value if decisions remain slow, unclear, untraceable, or disconnected from delivery.

Velocity exists to detect and correct that trap.

## Executive Dashboard

Recommended dashboard:

| Metric | Meaning |
|---|---|
| Decision cycle time | Time from decision need to approved decision. |
| Decision traceability coverage | Percentage of major decisions linked to artefacts and delivery actions. |
| Artefact usefulness score | Whether artefacts are used for decisions, not just stored. |
| Governance throughput | Number of decisions processed without rework. |
| PMO alignment score | Degree to which architecture decisions appear in delivery controls. |
| AI review compliance | Percentage of AI-assisted outputs reviewed by accountable practitioner. |
| Executive confidence | Sponsor confidence in architecture-driven execution. |

## Sample KPIs

- reduce architecture decision cycle time by 30%
- 90% of major delivery decisions linked to an artefact
- 80% of artefacts mapped to a live decision or control
- monthly executive decision dashboard published
- no AI-assisted output used without human review
- PMO RAID entries linked to architecture decisions
- design authority decision backlog visible and prioritised

## Workshop Questions

1. Which decisions take too long?
2. Which artefacts are produced but not used?
3. Can executives trace decisions into delivery?
4. Where does PMO status lack architecture context?
5. Which governance forums create value?
6. Which reviews slow delivery without improving outcomes?
7. How is AI-assisted work reviewed?
8. What would improve sponsor confidence?

## Velocity Interpretation

ACMM tells whether the architecture function is mature.

EA Metrics tell whether architecture creates value.

Velocity Decision Metrics tell whether architecture improves the speed, quality, traceability, and execution of enterprise decisions.
