# Velocity Architecture Charter

## Mission

Public professional framework and standards authority.

## Owned Capabilities

- enterprise design
- enterprise architecture
- business architecture
- solution architecture
- architecture governance
- public reference models
- master templates
- methodology
- international, Australian, New Zealand and APAC alignment guidance
- framework-neutral evidence baseline
- public thought leadership

## Explicit Exclusions

- client-specific artefacts
- runtime policy enforcement
- credentials
- private Ordo Animi implementation
- claims of universal certification or automatic compliance

## Accountable Owner

Founder / CTO.

## Upstream Relationships

- magister-automatorum

## Downstream Relationships

- fabrica
- probatio
- ordo-animi

## Authority Boundaries

Authoritative for professional method and public framework standards. It must not absorb responsibilities assigned to other repositories.

## Stop Conditions

Stop and request Founder / CTO routing if a change would cross repository boundaries, introduce runtime implementation during v0.1, add credentials, claim deployment or assurance without evidence, or alter institutional ownership.

## Criteria For Future Implementation

Future implementation requires approved scope, evidence of need, repository-boundary review, security review, data classification, and explicit release acceptance.
