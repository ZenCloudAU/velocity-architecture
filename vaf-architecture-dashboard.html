Nine Production Best Practices

Applied to VAF

  1. Tool Calls Over MCP: Direct GitHub API calls for artefact commits, not MCP abstraction. Deterministic, auditable, fast.
  2. Direct Function Calls: Commits, timestamps, API posts handled in orchestration layer, not agent tool calls. Cheaper, faster, testable.
  3. Single Tool Per Agent: One agent = one responsibility. VAF Framework Validator, Artefact Generator, GitHub Publisher separate.
  4. Single-Responsibility Agents: Generation ≠ validation ≠ publishing. Split concerns cleanly to reduce non-determinism.
  5. Externalize Prompts: VAF governance rules, decision frameworks, artefact templates stored in GitHub. Non-technical stakeholders can iterate.
  6. Responsible AI Consortium: Generate EA artefacts with Claude + GPT-4 in parallel. Reasoning agent consolidates per VAF rules.
  7. Separate Workflow from MCP: VAF engine runs as REST API. MCP server thin adapter only. Independent scaling and iteration.
  8. Containerized Deployment: Docker image → Kubernetes. Phase 1 (your account), Phase 2 (client multi-tenancy via env vars).
  9. KISS Principle: Flat, readable, function-driven design. Agents do one job well. Orchestration transparent.
