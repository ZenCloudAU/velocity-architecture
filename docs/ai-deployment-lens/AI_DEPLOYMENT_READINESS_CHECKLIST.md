# AI Deployment Readiness Checklist

## Business and Value

- [ ] Business owner is named.
- [ ] Problem and intended outcome are clear.
- [ ] Users and affected processes are identified.
- [ ] Success measures are defined.
- [ ] Non-goals and stop conditions are explicit.

## Architecture

- [ ] Use case type is classified.
- [ ] Enterprise and solution dependencies are understood.
- [ ] Integration pattern is defined.
- [ ] Non-functional requirements are captured.
- [ ] Architecture decision record is approved.

## Data

- [ ] Data owner is named.
- [ ] Data classification is complete.
- [ ] Approved processing environment is confirmed.
- [ ] Provider retention and training terms are reviewed.
- [ ] Restricted or prohibited data is excluded.

## Model and Tool

- [ ] Alternatives were compared.
- [ ] Cheapest capable option was considered.
- [ ] Quality limitations are documented.
- [ ] Access, retention, residency, and audit controls are acceptable.
- [ ] Exit and replacement path is understood.

## Human Accountability

- [ ] Human reviewer is named.
- [ ] Approval boundary is documented.
- [ ] Prohibited autonomous actions are listed.
- [ ] Escalation path exists.
- [ ] Final accountability remains with a human role.

## Agent and Workflow Controls

- [ ] Tool permissions are least-privilege.
- [ ] Read, draft, write, publish, send, and deploy permissions are separately defined.
- [ ] Failures are logged and visible.
- [ ] Retry and fallback behaviour is documented.
- [ ] Morning or operational review queue is defined where applicable.

## Security, Risk, and Compliance

- [ ] Risk register is complete.
- [ ] Security review is complete.
- [ ] Privacy, legal, regulatory, and contractual checks are complete.
- [ ] Monitoring and incident response are defined.
- [ ] Shutoff or rollback mechanism exists.

## FinRev DevOps

- [ ] Expected token, platform, and infrastructure cost is estimated.
- [ ] Budget owner is named.
- [ ] Value measure is defined.
- [ ] Repository and deployment source of truth are identified.
- [ ] Build, test, deploy, monitor, and retire flow is documented.

## Operations

- [ ] Support owner is named.
- [ ] Operational metrics are defined.
- [ ] Review cadence is agreed.
- [ ] Material model/tool changes trigger reassessment.
- [ ] Retirement criteria are documented.

## Decision

- [ ] Approved for discovery only
- [ ] Approved for pilot with conditions
- [ ] Approved for deployment
- [ ] Not approved
