# Data Rules

## Permitted Data

- Repository-specific constitutional documentation
- Governance notes and placeholder examples
- Public or internal doctrine appropriate to Velocity Architecture

## Prohibited Data

- Real client data unless a future approved release explicitly allows it
- Personal information unless explicitly approved and classified
- Credentials, secrets, tokens, API keys or production identifiers
- Data owned by another repository

## Classifications

Expected classifications are Public, Internal, Confidential and Restricted.

## Client Data

Client data is not permitted in v0.1. Repository names do not establish data residency.

## Personal Information

Personal information is not permitted in v0.1 except public maintainer identity already present in normal Git history.

## Authority

Authoritative for professional method and public framework standards. This repository is authoritative only for its stated role and referential for other repository concerns.

## Storage And Processing

Current storage is Git repository documentation only. Current processing is manual review only. No automated write integration exists in v0.1.

## Retention And Deletion

Retain constitutional records while they are current. Delete misplaced sensitive data immediately through approved incident handling.
