# Architecture Decision Record (ADR)

**VAF VP3 — Truth Viewpoint | Technical Architecture Layer**

> One ADR = one technical architectural decision.  
> Once accepted: immutable. To change: supersede with a new ADR.  
> Stored in: version control, inside the codebase.

---

## ADR Identification

| Field | Value |
|-------|-------|
| **ADR ID** | ADR-NNN |
| **Title** | Short, descriptive name (one line) |
| **Date** | YYYY-MM-DD |
| **Owner** | Name and role |
| **Status** | Proposed \| Accepted \| Superseded \| Deprecated |
| **Supersedes** | ADR-[prior ID] or N/A |
| **Governing Decision** | TOM-[ID] — the Trade-off Matrix this implements |

---

## Context

> *Why does this decision need to be made?*  
> Describe the system context, the forces at play, and what makes this decision necessary.  
> Limit to what materially affects the choice.
