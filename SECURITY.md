# Security Policy

## Posture

This repository is private by default and governed by least privilege.

## Access Expectations

- Multi-factor authentication is expected for maintainers.
- Access should be limited to authorised contributors.
- Access should be removed when no longer required.

## Secret Handling

No secrets, credentials, tokens, API keys, certificates, private keys, passwords, connection strings or production identifiers may be committed. Examples must not contain credentials.

## Supply-Chain Restraint

No GitHub Actions, third-party dependencies, package managers or deployment configuration may be added during v0.1 unless a later release explicitly approves them.

## Incident Escalation

Suspected exposure, secret leakage, unauthorised access, or boundary violation must be escalated to the Founder / CTO. Do not disclose sensitive details in public channels.

## Change Review

Security-relevant changes require documented review. Future automation requires separate approval and must preserve the repository boundary.
