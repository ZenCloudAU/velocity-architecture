# Roadmap

## v0.1 Fundamentum

Doctrine and constitutional baseline for Velocity Architecture.

## v0.2 Recognitio

Codex consistency review of repository boundaries, files, data rules, interfaces, security posture and release evidence.

## v0.3 Constitutio

First bounded implementation or reusable scaffold, only after Founder / CTO approval and repository-boundary review.

## Later Directions

- Harden repository-specific governance.
- Add approved schemas only when real interfaces require them.
- Add implementation only after evidence, security review and release acceptance.

## Implementation Gates

No implementation begins without approved scope, data classification, security review, interface review and acceptance criteria.

## Explicit Exclusions

No application build, deployment, workflow, dependency, authentication, storage service, external integration or compliance claim is introduced by v0.1.
